1 Что такое Claude Code и зачем он нужен

Claude Code — это терминальная версия Claude, самая продвинутая из всех. Не веб-чат, не десктоп-приложение — именно терминал даёт доступ к функциям, которых нигде больше нет:

  • Управление файлами и папками на твоём компьютере
  • Выполнение команд в терминале
  • Подключение к внешним сервисам (Google, GitHub, Telegram)
  • Запуск мультиагентных команд
  • Полная автономная работа без твоего участия
Суть: ты превращаешь Claude из чат-бота в автономного исполнителя, который сам открывает файлы, парсит сайты, пишет код и отправляет отчёты.

2 Установка и первый запуск

Требования

Нужна платная подписка Pro или Max — на бесплатном тарифе Claude Code недоступен.

Linux: аналогично через терминал

Запуск

claude

Установи расширение claudecode в VS Code
Авторизуйся командой /login

3 Режим автономной работы

По умолчанию Claude спрашивает разрешение на каждый шаг — создать файл? подтверди. Выполнить команду? подтверди. Это неудобно.

Для полной автономии запускай с флагом:

claude --dangerously-skip-permissions
Предупреждение автора: используй осторожно — агент будет делать всё сам без вопросов. Убедись, что понимаешь, что он делает.

4 Файл CLAUDE.md — мозг проекта

Это самая важная концепция курса. CLAUDE.md — главная системная инструкция, которую агент читает перед началом любой работы в папке. Благодаря ему Claude сразу «знает» твой проект — не нужно каждый раз объяснять что и как.

Как создать

Создай файл CLAUDE.md в корне проекта. Формат .md (Markdown) — стандарт для AI-агентов.

Что писать внутри

# Project: Wells Industries — корпоративный сайт IT-компании

Tech Stack: React, Vite, Tailwind CSS

Design: Modern minimal, dark theme, orange accents

Sections:
- Hero: Abstract 3D shapes
- About: 5 years of automation experience
- Cases: Client success stories
- Contact: Form with validation

Agreements:
- Компоненты в camelCase
- Стили только через Tailwind классы
- Комментарии на русском языке
Важно: без CLAUDE.md Claude может предложить неверные библиотеки или сломать структуру проекта. С ним — даже в новом чате агент сразу вспомнит всё.

6 Управление контекстным окном

Контекст — это оперативная память Claude. Чем больше заполнено, тем дороже каждый запрос и тем хуже качество ответов.

Правило 1 — Одна задача, один чат

Никогда не смешивай разные задачи в одном диалоге. Закончил — открыл новый чат.

Правило 2 — Контроль порога 50%

Следи через /context. Если перевалило за 50–60% — используй /compact или /new.

Правило 3 — Ground First, Ask Second

Сначала попроси Claude изучить файлы/документацию, а затем вторым сообщением ставь задачу. Качество резко улучшается.

Правило 4 — Экономь на MCP

Каждый подключённый MCP-сервер (например, Google Sheets) съедает ~2000 токенов при каждом запросе. Если сервис нужен редко — лучше создай Skill.

8 MCP-серверы — подключение внешних сервисов

MCP (Model Context Protocol) — протокол, который объединяет API разных сервисов в единую схему. Автор сравнивает его с USB-хабом: воткнул один штекер — получил доступ к 38 инструментам Google Sheets сразу.

Сервисы из видео

Google Sheets Google Drive Google Calendar Gmail Chrome DevTools GitHub Slack Notion Asana RSS-парсинг

Chrome DevTools позволяет автоматизировать браузер — Claude может зайти на Wildberries, найти товар и добавить в корзину.

Как подключить

Способ 1 — через агента (проще)

Скопируй команду-промт с маркетплейса (например, smizery.com) и вставь в чат Claude Code. Агент сам скачает и установит сервер.

Способ 2 — вручную

Через терминал с помощью npm или специфических скриптов. После установки Claude выдаст ссылку для авторизации — подтверди в браузере.

Конфигурация

Все настройки MCP хранятся в settings.json. Просмотр установленных серверов:

/mcp

9 Навыки (Skills) — многоразовые рабочие процессы

Skills — готовые пакеты экспертизы, которые обучают Claude делать повторяющиеся задачи определённым образом. В отличие от MCP, навыки хранятся локально в папке .claude/skills и включают инструкции, скрипты (Python, JS) и шаблоны.

Как создать навык

Способ 1 — Skill Creator

Специальный официальный навык от Anthropic. Он «интервьюирует» тебя — спрашивает о целях, входных данных и желаемом формате — и сам пишет файл skill.md.

Способ 2 — из диалога

Если Claude дал идеальный результат в чате — напиши: «Преврати это в навык».

Структура файла skill.md

---
name: humanizer
description: Переписывает ИИ-текст, делая его человечным
---

[Инструкция для агента...]
Описание (description) — критически важно. По нему Claude понимает, когда автоматически активировать навык.

10 Agent Teams — мультиагентные системы

Это главная «суперсила» Claude Code. Для сложных задач Claude автоматически запускает субагентов в изолированных сессиях — каждый со своим чистым контекстом и конкретной ролью.

Как это работает технически

Основной агент получает задачу
Он открывает отдельные сессии для субагентов
Каждый субагент работает изолированно — не перегружает общий контекст
Результаты собираются и объединяются

Инструкции для субагентов хранятся в .claude/agents/ в формате Markdown с YAML-заголовком:

---
name: scraper
description: Парсит данные с веб-сайтов
tools: browser, search
model: claude-sonnet-4-6
---

[Инструкция агента...]

Стандартная команда агентов

РольЗадача
Router / TeamLeadКоординатор: анализирует запрос, решает кого вызвать и в каком порядке
ScraperСобирает данные с сайтов
AnalystПроводит глубокое исследование
Report GeneratorФормирует отчёты в PDF, Excel, HTML

Включение режима Agent Teams

В settings.json добавить:

{
  "experimental_agent_teams": true
}

Это позволяет агентам общаться и обмениваться контекстом между собой, а не только работать изолированно.

Tmux — наблюдение за агентами

Установи Tmux — программу для разделения терминала на окна. Видишь работу всех агентов одновременно в реальном времени.

12 Практический пример — Автономный финансовый анализ

Вот что система делает полностью самостоятельно по одному запросу:

Парсинг данных
Scraper через Chrome DevTools MCP заходит на сайты, делает скриншоты, считывает котировки.

Итог: конкретные рекомендации — «держать рубль» / «покупать доллар» — на основе десятков источников. Без твоего участия.

16 Hooks и Settings

Файл settings.json

Бывает двух видов:

  • Глобальный — настройки для всего компьютера
  • Локальный — настройки только для конкретного проекта

Hooks — триггеры на события

Специальные обработчики, которые срабатывают на события. Автор настраивает их для уведомлений:

  • Terminal Bell — звуковой сигнал когда Claude закончил долгую задачу
  • Сигнал на паузу — звук когда агент ждёт твоего подтверждения
Deny Permissions — защита .env файла

Обязательно пропиши в настройках запрет на чтение файла .env, где хранятся пароли и API-ключи — чтобы AI случайно не отправил их в облако.

{
  "deny_permissions": [".env"]
}

Настройка через /config

Команда /config открывает меню настроек, где можно:

  • Включить режим размышления (thinking mode)
  • Настроить уведомления и хуки
  • Включить автоматическое сжатие контекста
  • Настроить .gitignore — чтобы Claude не индексировал лишние папки (например, node_modules)

17 Память и работа с кодом

Memory — постоянная память

В настройках Claude можно включить функцию памяти — агент будет запоминать информацию о тебе (возраст, предпочтения, стек технологий) и не переспрашивать это в новых чатах.

Vibe Coding — советы по работе с кодом

Главное правило — файлы до 600 строк

Держи файлы короткими — менее 600 строк. Большие файлы заставляют AI «галлюцинировать» и допускать ошибки.

Правильный порядок работы с проектом
Надиктуй «хотелки» в файл instructions.md
Попроси Claude составить implementation plan (план реализации)
Попроси создать CLAUDE.md на основе плана
Только после этого — запускай кодинг

/rewind — откат состояния файлов

Команда /rewind откатывает не только переписку, но и физически возвращает файлы к предыдущему состоянию. Если Claude сломал код — файлы вернутся к чекпоинту как будто ничего не было.

21 Промт-формулы автора

Три конкретные техники, которые Никита Велс использует в работе.

Формула ICC

Instructions + Context + Constraints

  • Instructions — что именно нужно сделать
  • Context — зачем, для кого, в каком окружении
  • Constraints — ограничения: формат, стек, запреты
Ground First, Ask Second

Шаг 1: попроси Claude изучить файлы, документацию или код — «заземлиться» в контексте.

Шаг 2: только вторым сообщением ставь задачу.

Качество ответа резко возрастает, потому что у модели уже есть полный контекст перед тем как она начинает думать над решением.

Context Interview

В конце любого промта добавляй фразу:

"Задай мне уточняющие вопросы, которые тебе нужны для лучшего выполнения этой задачи"

Это гарантирует что AI получит весь необходимый контекст перед началом работы — вместо того чтобы угадывать.

22 Экономия кредитов — 8 способов сэкономить 80%

Создай файл settings.json.local и пропиши API-ключ от openrouter.ai. Это позволяет подключать бесплатные модели (Nvidia, Minimax и другие) прямо в Claude Code.

{
  "api_key": "sk-or-ваш-ключ",
  "api_url": "https://openrouter.ai/api/v1"
}

25 Нюансы разработки с Claude Code

Стратегия микро-билдинга

Правило: создал одну мелкую фичу → протестировал → зафиксировал в Git. Никогда не давай Claude делать несколько фич за раз. Если что-то сломается — откатишься к последнему чекпоинту, а не к началу.

Ask User Question — принудительное интервью

Инструмент, который запускает диалог перед кодингом. AI сам задаёт уточняющие вопросы — стек, фреймворк, ограничения (например: «Next.js или Vite? Tailwind или CSS Modules?»). Запускается командой или добавлением в промт.

Race Conditions — ошибка которую AI не найдёт сам

Состояние гонки (race condition) — критическая ошибка при параллельной работе нескольких агентов с общими данными. Claude не исправит её автоматически — нужно явно попросить: «Проверь код на наличие race conditions».

Gedra MCP — реверс-инжиниринг

Специальный MCP-сервер для анализа бинарных файлов и реверс-инжиниринга. Позволяет Claude разбирать скомпилированные программы без исходного кода.

32 Plan Mode — архитектура без кода

Shift+Tab — включить Plan Mode
Описываешь задачу — Claude строит архитектурный план
Согласовываешь или правишь план
Выходишь из Plan Mode — Claude реализует по плану
